  DEBUGGING DE RESERVAS - Guía para verificar el bloqueo de 2 horas
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA REPORTADO:
  "Hice una reserva para mañana a las 8 AM, pero cuando vuelvo a hacer otra,
   sigue apareciendo disponible"

═══════════════════════════════════════════════════════════════════════════════
🔍 CÓMO VERIFICAR QUÉ ESTÁ PASANDO
═══════════════════════════════════════════════════════════════════════════════

PASO 1: Abre DevTools
  • Presiona: F12 (o Ctrl+Shift+I)
  • Ve a tab: "Console"

PASO 2: Haz una reserva completamente
  • Nombre: "Test User"
  • Email: test@example.com
  • Teléfono: 555-1234
  • Servicio: (elige uno)
  • Fecha: MAÑANA (la que aparezca por defecto)
  • Hora: 8:00 AM
  • Envía

  En la consola deberías ver:
    ✅ RESERVA CONFIRMADA
    ✅ Reserva agregada: 2026-03-18 08:00-10:00 (Test User)

PASO 3: Escribe en consola para verificar el estado
  • Escribe: debugReservas()
  • Presiona Enter

  Deberías ver algo como:
    📊 ===== ESTADO ACTUAL DEL SISTEMA DE RESERVAS =====
    📋 RESERVAS REGISTRADAS:
       1. 2026-03-18 08:00-10:00 (Test User)
    📆 DÍAS OCUPADOS:
       (ninguno)

PASO 4: Verificar qué horas están ocupadas
  • Obtén la fecha que se reservó (en este caso: 2026-03-18)
  • Escribe en consola: verificarHorasOcupadas("2026-03-18")
  • Presiona Enter

  Deberías ver:
    📊 ESTADO PARA: 2026-03-18
    Total horarios: 7
    Disponibles: 5
    ⏸️ Horas ocupadas:
       ❌ 08:00
       ❌ 10:00
    ✅ Horas disponibles:
       ✓ 12:00 PM
       ✓ 2:00 PM
       ✓ 4:00 PM
       ✓ 6:00 PM
       ✓ 8:00 PM

PASO 5: Vuelve a hacer otra reserva SIN recargar la página
  • Rellena formulario nuevamente
  • Selecciona MISMA FECHA (2026-03-18)
  • El select de hora debe mostrar ahora:
    ❌ 8:00 AM (OCUPADA) - deshabilitada en rojo
    ❌ 10:00 AM (OCUPADA) - deshabilitada en rojo
    ✓ 12:00 PM - normal
    ✓ 2:00 PM - normal
    etc.

═══════════════════════════════════════════════════════════════════════════════
⚠️ POSIBLES PROBLEMAS Y SOLUCIONES
═══════════════════════════════════════════════════════════════════════════════

PROBLEMA 1: No veo las horas ocupadas en el select
──────────────────────────────────────────────────
Verificación:
  1. Ejecuta: debugReservas()
  2. Verifica que ves la reserva en "RESERVAS REGISTRADAS"
  3. Ejecuta: verificarHorasOcupadas("2026-03-18")
  4. Verifica que dice "Ocupadas: 08:00, 10:00"

Si ves la reserva pero no en el select:
  → Recarga la página (F5)
  → Los datos en memoria se pierden
  → Esto es NORMAL - las reservas no se guardan en base de datos

Si NO ves la reserva en debugReservas():
  → Verifica que enviaste el formulario correctamente
  → Debiste ver "✅ RESERVA CONFIRMADA" en consola
  → Si no viste ese mensaje, la reserva no se registró


PROBLEMA 2: La hora no se deshabilita aunque está en horasOcupadas
──────────────────────────────────────────────────────────────────
Verificación:
  1. Ejecuta: debugReservas()
  2. Copia la fecha exacta que veas (ej: "2026-03-18")
  3. Selecciona esa fecha en el formulario
  4. Ejecuta: verificarHorasOcupadas("2026-03-18")
  5. Ver la consola cuando se actualiza el select
     (debería ver log: "🕐 2026-03-18: X/Y horas disponibles")

Si la fecha es diferente:
  → Problema: Seleccionaste diferente fecha en cada reserva
  → Solución: Selecciona la MISMA fecha en ambas reservas


PROBLEMA 3: Recargué la página y desaparecieron las reservas
───────────────────────────────────────────────────────────
Esto es ESPERADO:
  • Las reservas se guardan EN MEMORIA
  • Si recarga, se pierden
  • Para persistencia, se necesaría localStorage o base de datos
  • POR AHORA es normal mientras estés testeando

🧪 CASOS DE PRUEBA PASO A PASO
═══════════════════════════════════════════════════════════════════════════════

TEST 1: Bloqueo correcto de 2 horas
──────────────────────────────────────

1. Abre DevTools → Console
2. Haz reserva:
   Fecha: 18/03/2026
   Hora: 08:00

3. Consola muestra: ✅ Reserva agregada: 2026-03-18 08:00-10:00

4. Ejecuta: verificarHorasOcupadas("2026-03-18")

5. Verifica que aparezcan ambas horas:
   ❌ 08:00 (reservada)
   ❌ 10:00 (bloqueada automáticamente)

RESULTADO ESPERADO: ✅ ÉXITO


TEST 2: Actualización de select después de reserva
───────────────────────────────────────────────────

1. Ejecuta reserva (Test 1)
2. En consola deberías ver:
   "Selecciona una fecha" (reset automático)
3. Hace clic en calendario para seleccionar la MISMA fecha
4. El select debe mostrar:
   • 08:00 AM (OCUPADA) - deshabilitada
   • 10:00 AM (OCUPADA) - deshabilitada
   • 12:00 PM - normal
   • etc.

RESULTADO ESPERADO: ✅ ÉXITO


TEST 3: Múltiples reservas mismo día
────────────────────────────────────

1. Reserva 1: 2026-03-18 08:00 (bloquea 08:00-10:00)
2. Reserva 2: 2026-03-18 14:00 (bloquea 14:00-16:00)
3. Reserva 3: 2026-03-18 18:00 (bloquea 18:00-20:00)

4. Ejecuta: verificarHorasOcupadas("2026-03-18")

5. Debes ver 6 horas ocupadas (3 bloques de 2 horas):
   ❌ 08:00, 10:00, 14:00, 16:00, 18:00, 20:00

6. Solo disponibles: 12:00 PM

RESULTADO ESPERADO: ✅ ÉXITO


TEST 4: Bloquea conflicto de hora siguiente
──────────────────────────────────────────

1. Haz reserva: 2026-03-18 08:00
   (bloquea 08:00-10:00)

2. Intenta hacer reserva: 2026-03-18 10:00
   (debería estar bloqueada)

3. El select NO mostrará "10:00 AM" como seleccionable
   (aparecerá "10:00 AM (OCUPADA)" - deshabilitada)

RESULTADO ESPERADO: ✅ ÉXITO (no puede seleccionar 10:00)

📋 FUNCIONES DE DEBUGGING DISPONIBLES
═══════════════════════════════════════════════════════════════════════════════

1. debugReservas()
   ├─ Muestra todas las reservas actuales
   ├─ Muestra los días marcados como ocupados
   └─ Uso: debugReservas()

2. verificarHorasOcupadas(fechaISO)
   ├─ Muestra horas bloqueadas para una fecha específica
   ├─ Muestra horas disponibles para esa fecha
   └─ Uso: verificarHorasOcupadas("2026-03-18")

3. SISTEMA_DISPONIBILIDAD.reservas
   ├─ Array completo de reservas
   └─ Uso en consola: SISTEMA_DISPONIBILIDAD.reservas

4. SISTEMA_DISPONIBILIDAD.getHorasOcupadas(fechaISO)
   ├─ Retorna array de horas ocupadas
   └─ Uso: SISTEMA_DISPONIBILIDAD.getHorasOcupadas("2026-03-18")
